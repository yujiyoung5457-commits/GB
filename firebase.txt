Firebase 및 Zustand 연동 변경 기록
===================================

1. Firebase 초기화
- src/firebase.js를 추가했습니다.
- .env의 VITE_FIREBASE_* 환경변수로 Firebase Authentication과 Firestore를 초기화합니다.
- 실제 키가 들어 있는 .env는 .gitignore에 추가했고, 변수 이름만 담은 .env.example을 만들었습니다.

2. Zustand 로그인 상태 관리
- src/store/useAuthStore.js를 추가했습니다.
- Firebase onAuthStateChanged로 새로고침 후에도 로그인 상태를 복원합니다.
- user, loading 상태와 initializeAuth, setUser, logout 기능을 제공합니다.
- src/App.jsx에서 앱 시작 시 인증 상태 감지를 시작합니다.

3. Firebase Authentication
- src/pages/SignUp.jsx: 이메일/비밀번호 회원가입과 닉네임 프로필 저장을 연결했습니다.
- src/pages/Login.jsx: 이메일/비밀번호 로그인을 연결했습니다.
- src/companent/Header.jsx: 로그인 상태에서는 기존 LOGIN 위치가 LOGOUT으로 동작합니다.
- 기존 Auth.module.scss와 인증 화면 구조는 변경하지 않았습니다.

4. Firestore 방명록 CRUD
- src/services/guestBookService.js를 추가했습니다.
- guestbooks 컬렉션에서 실시간 조회(onSnapshot), 작성(addDoc), 수정(updateDoc), 삭제(deleteDoc)를 처리합니다.
- 문서에는 authorId, authorEmail, nickName, message, character, createdAt, updatedAt을 저장합니다.
- 기존 이모지 배열과 캐릭터 이미지 데이터는 유지했습니다.
- 기존 작성 폼과 좌우 배치는 유지하고, 아래에 방명록 목록만 추가했습니다.
- 비회원은 목록을 읽을 수 있지만 작성할 수 없습니다.
- 수정/삭제 버튼은 로그인한 UID가 문서의 authorId와 일치할 때만 표시됩니다.

5. Firebase 보안 규칙
- 프로젝트 루트의 firestore.rules에 전체 규칙을 작성했습니다.
- 모든 사용자는 guestbooks 문서를 읽을 수 있습니다.
- 로그인 사용자만 자신의 UID로 문서를 작성할 수 있습니다.
- 작성자 본인만 메시지, 캐릭터, updatedAt 필드를 수정할 수 있습니다.
- 작성자 본인만 자신의 문서를 삭제할 수 있습니다.
- guestbooks 외 모든 Firestore 경로는 기본 거부합니다.
- firebase.json이 firestore.rules를 배포 대상으로 지정합니다.

6. Firebase 콘솔 및 배포 시 필요한 작업
- Firebase Console > Authentication > Sign-in method에서 이메일/비밀번호 로그인을 활성화합니다.
- Firebase Console > Firestore Database에서 데이터베이스를 생성합니다.
- Firebase CLI 로그인과 프로젝트 연결 후 `firebase deploy --only firestore:rules`로 규칙을 배포합니다.
- 보안 규칙을 배포하기 전에는 콘솔에 설정된 기존 규칙이 적용되므로 반드시 배포해야 합니다.

7. 무료 Spark 프로젝트 규칙 배포 완료
- Firebase CLI를 개발 의존성으로 설치했습니다.
- Gemini 기능과 CLI 사용량 보고는 활성화하지 않았습니다.
- 데이터베이스가 freeTier인 것을 확인했습니다.
- Hosting, Functions, Storage는 배포하지 않고 Firestore 보안 규칙만 배포했습니다.
- permission-denied를 해결하여 SAVE 후 find e-mail 아래 실시간 카드 조회가 가능해졌습니다.
